## Overview
This project analyzes system-level security implementation for BioGenX following the Phantom Phoenix APT intrusion. The focus is on applying operating system hardening, automating security policy enforcement, evaluating policy implementation at the OS level, and assessing residual vulnerability exposure after hardening.

## Objectives
- Extrapolate a systems hardening implementation plan from cyber resilience strategies
- Apply systems hardening through scripted automation and policy configuration
- Identify security policies that should be prioritized for automation across platforms
- Evaluate operating system level policy implementation for risk reduction and compliance
- Assess residual vulnerabilities using vulnerability scanning, penetration testing, and risk communication

## Key Analysis Areas

### 1. Systems Hardening Implementation
- Established secure hardening baselines for manufacturing systems
- Reduced exposure by disabling unnecessary services and enforcing secure configurations
- Applied OT security protections through segmentation between enterprise IT and operational technology environments
- Integrated risk based decision making to prioritize critical manufacturing systems and remote administration assets
- Used continuous monitoring and maintenance to validate hardening effectiveness over time

### 2. Policy Automation Priorities
- Security auditing and logging
- Configuration management and baseline enforcement
- Patch management and software update policies
- Access control and privilege management
- Continuous monitoring and policy validation

### 3. Operating System Level Policy Implementation
- Enforced secure baseline configurations across systems
- Strengthened auditing and logging for visibility into logon, logoff, and policy change events
- Applied role based access controls to support least privilege
- Balanced security requirements with operational efficiency through granular policy design, staged deployment, and user acceptance considerations

### 4. Residual Vulnerability Exposure
- Evaluated remaining weaknesses after hardening through vulnerability scanning
- Assessed exploitability and control effectiveness through penetration testing concepts
- Reviewed monitoring alerts, system logs, and incident response metrics for evidence of remaining risk
- Applied structured risk communication to translate technical findings into business impact
